# PRD - סוכן AI לניהול הזמנות פגישה (Gmail + Calendar)

**קבוצה:** biu-len05 | **תרגיל:** ex03
**קורס:** מערכות בינה מלאכותית מתקדמות: מבוא ל-LLMs, סוכנים וריבוי סוכנים

## 1. רקע ובעיה

בהתאם לעקרונות שנלמדו בהרצאה L06 (MCP & API for Agents), פרויקט זה בונה סוכן AI שמתחבר לשני שירותי Google - Gmail ו-Calendar - ומבצע באופן אוטומטי (למחצה) את תהליך ניהול הזמנות לפגישה: זיהוי הזמנה במייל, בדיקת זמינות ביומן, ותגובה מתאימה (יצירת אירוע או מייל דחייה).

## 2. מטרות

1. להדגים חיבור סוכן AI לשירותים חיצוניים באמצעות Skills העוטפים קריאות API (Gmail API, Calendar API עם אימות OAuth 2.0).
2. לממש זיהוי וחילוץ מידע מטקסט חופשי (הזמנת פגישה) באמצעות LLM.
3. לממש לוגיקת החלטה (זמין/תפוס) ותגובה אוטומטית בהתאם.
4. לתעד את עקרונות אבטחת המידע שנלמדו (Environment Variables, .gitignore, ללא Secrets בקוד).

## 3. היקף (Scope)

### בתוך ההיקף:
- אימות מול Google באמצעות OAuth 2.0 (Desktop app flow)
- קריאת מיילים מתיבת דואר מוגבלת (לדוגמה: N ימים אחרונים, או תיקייה מתויגת)
- זיהוי הזמנות לפגישה בטקסט חופשי (לא רק Calendar Invite פורמלי)
- חילוץ פרטי פגישה (תאריך, שעה, משתתפים, מקום) באמצעות LLM
- בדיקת זמינות ביומן Google Calendar
- יצירת אירוע חדש אם פנוי
- שליחת מייל תשובה אוטומטי אם תפוס
- תיעוד מלא של תהליך ההרשאות וה-Setup ב-README

### מחוץ להיקף:
- MCP מלא (שרת MCP נפרד) - עשוי להתווסף כהרחבה אם יישאר זמן
- תמיכה במספר תיבות דואר / חשבונות משתמש בו-זמנית
- ממשק משתמש גרפי

## 5. תרחיש בדיקה

1. שליחת מייל בדיקה עצמי עם טקסט חופשי המזמין לפגישה (למשל: "אפשר להיפגש ביום שלישי הקרוב בשעה 14:00 במשרד לדבר על הפרויקט?")
2. הרצת הסוכן וידוא שהוא מזהה את ההזמנה, מחלץ את הפרטים נכון
3. בדיקת שני מסלולים: יומן פנוי (יצירת אירוע) ויומן תפוס (שליחת מייל דחייה)

## 6. קריטריוני הצלחה

- הסוכן מזהה הזמנת פגישה בטקסט חופשי בעברית ו/או אנגלית
- חילוץ הפרטים מדויק ועקבי
- בדיקת הזמינות נכונה מול היומן האמיתי
- אין דליפת סודות (API keys, tokens) לריפו הפומבי

## 7. סיכונים וידועים

- OAuth consent screen במצב Testing - מוגבל למשתמשי בדיקה שנרשמו (זה תקין ומספיק למטלה)
- Refresh token עלול לפוג אחרי 7 ימים במצב Testing/External - נדרש לתעד זאת ולתכנן הרצה קרוב למועד ההגשה
- זיהוי הזמנת פגישה בטקסט חופשי הוא משימה לא טריוויאלית - ייתכנו false positives/negatives, יש לתעד מגבלה זו
